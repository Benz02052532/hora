# HORA — ผังองค์กร × โหราศาสตร์ไทย

เว็บหน้าเดียวสำหรับดูผังองค์กรบริษัท พร้อมเก็บข้อมูลโหราศาสตร์และรูปดวงของพนักงานแต่ละคน
พื้นหลังเป็นท้องฟ้าดาวจริงเหนือประเทศไทย ณ เวลาปัจจุบัน

## ความสามารถ

- **ผังองค์กรต้นไม้แนวตั้ง** — เพิ่มแผนกได้อิสระ ไม่จำกัดจำนวนชั้น
- **ย้ายตำแหน่งได้ 2 วิธี** — ลากวางบนคอมพิวเตอร์ หรือเลือกจาก dropdown บนมือถือ
- **ข้อมูลโหราศาสตร์** — วัน/เดือน/ปีเกิด · เวลาเกิด · จังหวัดที่เกิด
- **อัปโหลดรูปดวง** — บีบอัดอัตโนมัติ กดขยายดูเต็มจอได้
- **ท้องฟ้าดาวจริง** — ตำแหน่งดาวคำนวณตามหลักดาราศาสตร์ พร้อมทางช้างเผือกและเส้นกลุ่มดาว
- **ดาวพระเคราะห์ทั้ง 9** — อาทิตย์ จันทร์ อังคาร พุธ พฤหัส ศุกร์ เสาร์ ราหู เกตุ
  คำนวณแบบนิรายนะ (ระบบที่โหราศาสตร์ไทยใช้)
- **สำรองข้อมูล** — ส่งออก/นำเข้าเป็นไฟล์ `.json` ได้

## เทคโนโลยี

ไม่มี build step ไม่มี framework — เปิดไฟล์ `index.html` ก็ทำงานได้เลย

| ส่วน | ใช้อะไร |
|---|---|
| ฐานข้อมูล | Supabase (แพ็กเกจฟรี) |
| เก็บรูปดวง | Supabase Storage |
| โฮสต์ | GitHub Pages |
| คำนวณดาว | เขียนเองใน `js/astro.js` ไม่พึ่ง API ภายนอก |

ถ้ายังไม่ได้ตั้งค่า Supabase ระบบจะเก็บข้อมูลใน `localStorage` ของเบราว์เซอร์ให้อัตโนมัติ

## โครงสร้างไฟล์

```
hora/
├── index.html          โครงหน้าเว็บ
├── config.js           ⚙️ ตั้งค่าทั้งหมดอยู่ที่นี่ไฟล์เดียว
├── css/style.css       ธีมดวงดาว
├── js/
│   ├── app.js          ผังองค์กร ฟอร์ม modal
│   ├── store.js        เชื่อม Supabase / localStorage
│   ├── astro.js        คำนวณดาราศาสตร์และโหราศาสตร์ไทย
│   ├── sky.js          วาดท้องฟ้าบน canvas
│   ├── stars.js        แคตตาล็อกดาวสว่างและเส้นกลุ่มดาว
│   └── wheel.js        วงล้อจักรราศี (SVG)
└── docs/schema.sql     SQL สร้างตารางบน Supabase
```

## การตั้งค่า

แก้ไฟล์ [`config.js`](config.js) ไฟล์เดียว:

```js
companyName: 'ชื่อบริษัทของคุณ',
password:    'รหัสผ่านเข้าเว็บ',
supabaseUrl: 'https://xxxxx.supabase.co',
supabaseAnonKey: 'sb_publishable_xxxxx',
latitude:  13.7563,   // พิกัดสำหรับคำนวณท้องฟ้า
longitude: 100.5018,
```

### ติดตั้ง Supabase ใหม่ตั้งแต่ต้น

1. สร้างโปรเจกต์ที่ [supabase.com](https://supabase.com) (เลือก region Singapore)
2. เปิด **SQL Editor** → วางเนื้อหาจาก [`docs/schema.sql`](docs/schema.sql) → Run
3. ไปที่ **Settings → API Keys** → copy `Publishable key`
4. เอา Project URL และ key ใส่ใน `config.js`

## ⚠️ ข้อจำกัดด้านความปลอดภัย

เว็บนี้ใช้**รหัสผ่านที่ฝังอยู่ในโค้ดฝั่งผู้ใช้** และ repo เป็นสาธารณะ

**แปลว่าใครก็ตามที่เปิดดู source code จะเห็นรหัสผ่านและ Supabase key**
และดึงข้อมูลวันเกิดของพนักงานได้โดยตรง แม้ไม่ผ่านหน้า login

เป็นการแลกความสะดวกกับความปลอดภัยโดยตั้งใจ **อย่าเก็บข้อมูลที่อ่อนไหวกว่านี้ในระบบนี้**

ถ้าต้องการความปลอดภัยจริง ให้เปลี่ยนไปใช้ Supabase Auth (บัญชีกลาง 1 บัญชี)
แล้วตั้ง RLS policy ให้จำกัดเฉพาะผู้ที่ล็อกอินแล้ว — ประสบการณ์ผู้ใช้ยังเหมือนเดิมคือกรอกรหัสเดียว

## หมายเหตุเรื่องข้อมูลส่วนบุคคล

วันเกิดและเวลาเกิดของพนักงานถือเป็นข้อมูลส่วนบุคคลตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)
ควรแจ้งวัตถุประสงค์และขอความยินยอมจากพนักงานก่อนจัดเก็บ

## ความแม่นยำของการคำนวณ

ตำแหน่งดาวคำนวณด้วยธาตุวงโคจรโดยประมาณ (ใช้ได้ช่วง ค.ศ. 1800–2050)
คลาดเคลื่อนระดับลิปดา — เพียงพอสำหรับบอกราศี แต่ไม่ควรใช้แทนตำราโหราศาสตร์ในการพยากรณ์จริงจัง
